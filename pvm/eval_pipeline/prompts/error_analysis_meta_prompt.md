## 지시사항

위 트레이스를 읽고 다음을 수행하세요.

**1단계: 실패 카테고리 귀납적 도출**
- Fail 케이스들의 human_reason을 읽고 공통 패턴을 찾아 {category_count_guidance}의 실패 카테고리를 도출하세요.
- 미리 정해진 목록을 만들지 말고, 실제 데이터에서 카테고리가 자연스럽게 드러나게 하세요.
- 표면적 유사성이 아닌 **근본 원인** 기준으로 분류하세요.
- 각 카테고리는 상호 배타적이어야 합니다.

**2단계: 각 카테고리에 대해 action 결정**
- `"judge_prompt"`: 의미적 이해가 필요해 LLM judge가 필요한 경우
- `"code_check"`: 정규식, 키워드 매칭, 파싱 등 코드로 체크 가능한 경우
- `"ignore"`: 발생 빈도가 낮고 비즈니스 영향이 미미한 경우

**3단계: 전체 트레이스 라벨링**
- 제공된 **모든** 트레이스(Pass 포함)에 대해 카테고리별 이진 라벨(Pass/Fail)을 부여하세요.
- 각 트레이스는 여러 카테고리에서 동시에 Fail일 수 있습니다.
