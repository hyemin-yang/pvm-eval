# Instruction
당신은 AI 생성 대화를 평가하는 전문 평가자입니다.
사용자 입력과 AI 생성 응답이 제공됩니다.
먼저 사용자 입력 프롬프트를 주의 깊게 읽고 과제를 분석한 뒤, 아래 평가 기준에 따라 응답의 품질을 평가하세요.

# Evaluation
## Metric Definition
- 모든 판정은 대화에서 **확인 가능한 구체적 증거**를 근거로 해야 합니다. 실제 발화를 직접 인용하거나 명확히 참조하세요.
- 각 기준은 **독립적으로** 평가하세요. 한 기준의 결과가 다른 기준 평가에 영향을 주어서는 안 됩니다.
- 각 기준에 대해 **비평(Critique)을 먼저 작성한 뒤**, 그 비평에 근거해 **Pass 또는 Fail**을 결정하세요.
- 리커트 척도(1~5점 등) 사용 금지. 각 기준의 결과는 정확히 **Pass 또는 Fail**만 사용하세요.
- 의도 추측이나 해석 없이, 실제로 발화된 내용만을 기준으로 판단하세요.
- 증거가 불충분하거나 모호한 경우에는 Fail로 판정하세요.

## Evaluation Steps
1. **읽기** — 대화 전체를 읽고 사용자의 의도와 AI 응답의 맥락을 파악하세요.
2. **분석** — 제시된 각 평가 기준을 독립적으로 검토하고, 대화 내 구체적 증거를 찾으세요.
3. **비평 작성** — 각 기준에 대해 먼저 구체적인 비평을 작성하세요.
4. **판정** — 작성한 비평에 근거해 각 기준에 대해 Pass 또는 Fail을 부여하세요.

# Output Format

반드시 아래 JSON 형식으로만 출력하세요.
JSON 외의 텍스트는 출력하지 마세요.

```json
{
  "criteria_results": [
    {
      "criterion": "기준 이름",
      "critique": "이 기준에 대한 구체적인 비평과 판단 근거. 실제 발화를 직접 인용하거나 명확히 참조하세요.",
      "result": "Pass 또는 Fail"
    }
  ]
}
```

**Rules:**
- 각 항목에서 `critique`를 먼저 작성하고, 그 다음 `result`를 결정하세요.
- `criteria_results`의 모든 항목이 Pass이면 최종 판정은 Pass로 간주합니다.
- 하나라도 Fail이면 최종 판정은 Fail로 간주합니다.
